Instructions

Please complete this project yourself. While you are encouraged to have high-level discussions with your fellow students, you must be able to complete the details of this project on your own.

Late Policy: All projects are due by 11:59 pm of the stated due date. Projects submitted after the due date/time are considered late and will not be graded.  It is your responsibility to submit projects on time.

Learning Objectives

After completing this project, you will:

  • Gain experience using git via your CLI and Visual Studio Code (VSCode) Source Control
  • Gain experience writing and executing non-web server Node.js JavaScript code
  • Practice refactoring JavaScript code

Overview

Project elements:

  • Use the command line interface (CLI) of your operating system to create and work with a git repository (repo)
  • Refactor the JavaScript program from the previous project to practice using git and practice refactoring
  • Use git via VSCode
    • Create and use a .gitignore file
    • Diff split screen

Requirements

Complete each of the following requirements.

Part 1. Completed Lab 2
>>: You should have completed Lab 2 and saved lab-02.zip in your cit281/p2 folder.

Part 2. Refactor and git
As part of Project 1 you created a file p1-random.js.

>>: You will use the Canvas posted solution for this file, but apply a few refactor changes to address the following goals:

  • Isolate code into functions
  • Eliminate global variables
  • Make code self-documenting by variable and function name choices

Use the CLI of your operating system to accomplish the following tasks:

>>: Create a new sub-folder in your cit281/p2 folder called refactor.

>>: Change to the cit281/p2/refactor folder.

>>: Initialize the cit281/p2/refactor folder as a git repository (repo).

>>: Create a new file, p2-random.js in the cit281/p2/refactor folder, using the Canvas posted solution for Project 1 as the initial source code of the file.

>>: Update the p2-random.js file documentation section of the new file with the correct project number and your name.

>>: Use VSCode to ensure the code runs with the expected output.

>>: Add the p2-random.js to git.

>>: Commit the p2-random.js to git with a documentation message, "First commit". You must include the documentation message.

>>: Create a new function getRandomLetter() that will return a single, random, lowercase letter. Note the following about this function:

  • This function will also include the alphabet constant.
  • Include a single line comment above the function that documents the purpose/functionality of the function.
  • Update the for loop to call getRandomLetter().

>>: Use VSCode to ensure the code runs with the expected output.

>>: Stage and commit the p2-random.js changes to git with a documentation message, "Added getRandomLetter()".

>>: Create a new function getRandomString(minLength, maxLength) that will return the random length string. Note the following about this function:

  • The maxLength parameter will be actual maximum string length, so adjust any getRandomInteger() parameters appropriately.
  • The function will call getRandomLetter().
  • Use a value of 10 for the minimum string length, and 20 for the maximum string length.
  • Update the console.log() to call getRandomString().
  • Include a single line comment above the function that documents the purpose/functionality of the function.

>>: Use VSCode to ensure the code runs with the expected output.

>>: Stage and commit the p2-random.js changes to git with a documentation message, "Added getRandomString()".

>>: Add a final function to p2-random.js called getSortedString(string) that will return a string in ascending order (i.e., xpacd would return acdpx). Note the following about this function:

  • The function body MUST be a single line of code. How? Consider array methods.
  • Include a single line comment above the function that documents the purpose/functionality of the function.

>>: Use VSCode to ensure the code runs with the expected output.

>>: Stage and commit the p2-random.js changes to git with a documentation message, "Added getSortedString()".

>>: Use the following command to list the files in the git repository. The output of the command will be saved to file p2-random-files.txt. The --stage option includes more details about the tracked files in the repo.

Tip: You may want to use your operating system CLI command to display the file contents (cat or type) to make sure the redirection of the git output was saved in the text file.

git ls-files --stage > p2-random-files.txt

>>: Use the following command to list a log of the repo changes. The output of the command will be saved to file p2-random-commits.txt. The --pretty=oneline option will produce a single line for each of the commits, showing the commit documentation message.

git log --pretty=oneline p2-random.js > p2-random-commits.txt

Part 3. Function expressions and VSCode git

Rather than a normal function, JavaScript frequently utilizes function expressions. Part of a function expression is an anonymous function. In the following sample code, the yellow portion is the anonymous function. Assigning a function to a variable is a function expression.

const varName = function() { }

You will be creating a new file, p2-expressions.js, in the cit281/p2/refactor folder, that replaces all of the function definitions with function expressions. Rather than use the CLI with git commands, this time we'll use VSCode to perform the git steps. 

Use VSCode to accomplish all of the following tasks, including git operators:

>>: Use VSCode Explorer tab to create a new file, p2-expressions.js, in the cit281/p2/refactor folder. Copy the content of p2-random.js into p2-expressions.js.

>>: Use the VSCode Source Control tab to stage the new file, p2-expressions.js, for use with git by highlighting or selecting the p2-expressions.js file, and clicking on the Stage icon (the plus symbol) to the right of the file. See the images below.

  

>>: Once staged, the file will be moved into a new section called Staged Changes, indicating the file is ready to commit. Select the staged file, then select the Commit icon (the check mark icon) to commit the staged file. Use the documentation message, "Filed added" and press the Enter key to complete the commit process. See the images below.

  

  

>>: Rewrite each function as a function expression using the pattern demonstrated above. Use the original function name as the function expression variable.

>>: Use VSCode to ensure the code runs with the expected output.

>>: Again use VSCode Source Control tab to stage and commit the change, using "Functions refactored into function expressions" as the documentation message.

Part 4. Ignoring files and diff

In VSCode the two text files are listed. We do not want to track text files for this repo, so we will use a git .gitignore file.

>>: Use VSCode Explorer to create a .gitignore file with the following contents

*.txt

>>: Switch to the VSCode Source Control tab and note that the previously displayed text files no longer appear, and now only the .gitignore file appears.

>>: Select the .gitignore file in the Source Control tab, and note in the source window to the right a split pane appears, showing the differences between what is currently in git, and what will be added to git, when the .gitignore file is added.

>>: Use VSCode Source Control to stage and commit the .gitignore file with a documentation message, "Added".

>>: Modify the .gitignore file by adding the following line:

*.bak

>>: Switch to the Source Control tab, select the file, and again note the diff split screen. Screen capture both sides of the diff split screen and save the screen capture as p2-vscode-diff.png.

>>: Use VSCode Source Control to stage and commit the .gitignore file with a documentation message, "Ignore .bak files".
